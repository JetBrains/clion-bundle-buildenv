# MinGW cross-compilation packages

PKGBUILD-based build of a MinGW-w64 toolchain (GCC, binutils, make, and dependencies).
Packages here are based on versions synced from [MSYS2 MINGW-packages](https://github.com/msys2/MINGW-packages)

Packages get cross-compiled for `x86_64-w64-mingw32` on Linux.

## Package layout

```
mingw/<package>/
  upstream.conf       # pinned MSYS2 commit + package path
  PKGBUILD.template   # cross-compile template with {{placeholders}}
  PKGBUILD            # generated via sync script — do not edit directly
  *.patch             # optional local patches (applied after upstream patches)
```

## Syncing to a newer upstream version

1. Find the desired commit in [MINGW-packages](https://github.com/msys2/MINGW-packages).
2. Update `upstream.conf` of the desired package with the following:
   ```
   UPSTREAM_COMMIT=<new-commit-hash>
   UPSTREAM_PACKAGE_PATH=mingw-w64-gcc   # rarely changes
   ```
3. Run sync:
   ```bash
   ./scripts/sync-mingw-packages.sh gcc          # single package
   # or 
   # ./scripts/sync-mingw-packages.sh --all        # all packages with upstream.conf
   ```
   This fetches the upstream PKGBUILD, extracts version/sources/patches,
   downloads patches, and renders `PKGBUILD` from the template.
4. Review the generated `PKGBUILD` diff and test the build.

## What to edit

- `upstream.conf` — to pin a different upstream commit
- `PKGBUILD.template` — to change cross-compilation logic (configure flags, sed fixups, install steps)

## What not to edit

- `PKGBUILD` — regenerated by `sync-mingw-packages.sh`; local edits will be overwritten
- patches fetched by `sync-mingw-packages.sh`

## Packages without upstream sync

Some packages have a `PKGBUILD` but no `upstream.conf` or template (e.g. `manifest`).
These are maintained manually. If you need to update them or change their build logic,
add a new PKGBUILD.template and upstream.conf to the package directory and run `sync-mingw-packages.sh {{package}}`.
